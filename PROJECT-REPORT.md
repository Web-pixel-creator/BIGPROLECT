# Отчет по проекту Bolt DIY (текущий статус)

## Назначение
Платформа для генерации UI (чат + конструктор), с автоподстановкой готовых компонентов/блоков из локальных библиотек (MD-файлы) и подключаемых реестров (MCP/registry).

## Ключевые интеграции и файлы
- `app/components/chat/ChatBox.tsx` — панель быстрых промптов/эффектов/секций/тем, предпросмотр реестров, ручное обновление реестров, закрытие по клику вне панели, индикатор статуса реестров.
- `app/lib/services/componentMatcher.ts` — поиск релевантных компонентов по промпту; поддержка алиасов/синонимов (RU/EN); вытягивает сниппеты из MD-файлов.
- `app/lib/constants/promptPresets.ts` — набор быстрых промптов/эффектов/секций/тем.
- `app/lib/services/registryService.ts` и `app/api/registry/api.registry.ts` — работа с registry (MCP/shadcn совместимые реестры), статус/кэш/превью.
- MD-библиотеки компонентов (локальный источник для componentMatcher):
  - `shadcnui-blocks.md`
  - `aceternity-components.md`
  - `kokonutui-components.md`
  - `magicui-components.md`
  - `reactbits-components.md`

## Что сделано
- Включена панель быстрых промптов в чате: подбор промптов/эффектов/секций/тем одним кликом, закрытие вне панели, индикатор реестров, кнопка «Обновить».
- Починены русские тексты и тултипы в панели/индикаторе реестров.
- componentMatcher читает все локальные MD-библиотеки и умеет маппить запросы по алиасам (глитч/aurora/plasma/… и др.).
- Реестры снова задействованы (registryService), есть превью первых элементов и ручное обновление.
- Добавлены крупные наборы компонентов в MD-файлы: shadcnui-blocks, Aceternity, KokonutUI, MagicUI, ReactBits.
- Настроены пресеты промптов/эффектов/секций/тем для быстрого старта.

## Как запустить
- Установить зависимости (если нужно): `pnpm install`
- Дев-сервер (Vite): `pnpm dev` (по умолчанию http://localhost:5173; можно `--host --port 5174`)
- Electron дев-режим (если требуется): `pnpm dev:electron` (см. скрипты в package.json)

## Рекомендации / следующие шаги
- Прогнать быстрый smoke-тест: открыть панель, выбрать промпт/эффект, проверить статус реестров и генерацию.
- Если реестры будут тормозить: добавить тайм-аут/кэш или переключатель «выкл.» в UI.
- Актуализировать/сгруппировать пресеты в `promptPresets.ts`, убрать дубли.
- При росте MD-файлов — подумать о периодическом автообновлении или дедупликации, иначе промпт-контекст будет раздуваться.
- Добавить e2e smoke (или unit) для componentMatcher/registryService, чтобы отслеживать регрессии в подборе сниппетов.
